# ZeroGTM Docs & GitHub Pages

This folder is the source for the **GitHub Pages** site: documentation plus SEO-optimized marketing (features, compare, pricing).

**Encoding:** All content files (Markdown, HTML, YAML) are UTF-8. Save with UTF-8 (no BOM) so special characters and symbols render correctly.

## Enable GitHub Pages

1. In your repo, go to **Settings → Pages**.
2. Under **Build and deployment**, set **Source** to **Deploy from a branch**.
3. Choose **Branch**: `main` (or your default branch), **Folder**: `/docs`.
4. Click **Save**. The site will be available at:
   - `https://<owner>.github.io/cold-email-ninja-app/`

GitHub builds the site with Jekyll. The `_config.yml` in this folder sets `baseurl: "/cold-email-ninja-app"` so links work correctly.

## Local preview

Optional: install Jekyll and run a local server.

```bash
cd docs
bundle install   # if you use the Gemfile
bundle exec jekyll serve
```

Then open `http://localhost:4000/cold-email-ninja-app/` (with baseurl).

## Structure

- **Marketing**: `index.md` (home), `features.md`, `compare.md`, `pricing.md`
- **Blog**: `_blog/` (e.g. `google-maps-b2b-leads-2026.md`) — permalink `/blog/:name/`
- **Docs**: `docs/getting-started.md`, `docs/mobile-app.md`, `docs/web-app.md`, `docs/enrichment-pipeline.md`, `docs/self-hosted.md`, `docs/api-keys.md`
- **Layout**: `_layouts/default.html` (nav, footer, meta, canonical)
- **SEO**: `robots.txt` (via `robots.md` + `_layouts/robots.txt`, hardcoded sitemap URL), sitemap via `jekyll-sitemap` plugin, JSON-LD (SoftwareApplication, FAQPage, BreadcrumbList), OG/Twitter meta, [SEO audit](docs/seo-audit.md)

## Verify after deploy

- **robots.txt**: `https://<owner>.github.io/cold-email-ninja-app/robots.txt` — should show `Allow: /` and `Sitemap: https://man0l.github.io/cold-email-ninja-app/sitemap.xml`.
- **sitemap.xml**: `https://man0l.github.io/cold-email-ninja-app/sitemap.xml` — generated by `jekyll-sitemap`; should list all pages with full URLs including baseurl.

## Plugins (GitHub Pages)

GitHub Pages supports `jekyll-sitemap`, `jekyll-seo-tag`, and `jekyll-feed`. They are listed in `_config.yml`; no need to add them to a Gemfile for the deployed site.
